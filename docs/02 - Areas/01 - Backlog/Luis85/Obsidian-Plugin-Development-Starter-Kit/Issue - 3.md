# Constraints & Requirements
## Prerequisites

- Obsidian > 1.9 installed
- Git installed
- Node installed
- GitHub Account
- Fork of the Starter Kit
- Obsidian Git Plugin installed
- Obsidian Hotreload Plugin installed


## Requirements

- New Projects must start in GitHub to satisfy versioning and traceability requirements
- Projects must be well documented to improve lessons learned over time
- Decisions and Requirements must be traceable to satisfy compliance constraints
- The solution must be easy to use to grow and retain a stable user base
- The solution must provide an out of the box environment to quickly start a new project
- The solution must enable automation workflows to support self documenting projects

## Constraints

- The solution must be build on Git
- The solution must be compatible with GitHub
- The solution must work as stand alone template for Obsidian without dependencies to GitHub
- Implemented Solutions must be traceable back to an Issue
- The solution must be build Obsidian first
- The Git Repo reflects the current solution
- The GitHub Issue Tracker Feature acts as Backlog for the Project
- The GitHub Pull Requests Feature is used as the starting point for Change Management
- The Solution must provide clear indicators showing the current State of the Project


## Conventions

- Commits must include an Issue Id
- Work must be done on a Feature Branch based on open Issues
- Merges must follow GitHub Pull Requests Workflow


## Documentation 

- The solution must provide documentation and insights about the what, how, and why of the current implementation
- The solution must provide documentation based on the PARA method
- The solution must provide user manuals following the Diataxis format
- The solution must provide documentation following best practices based on Arc42 and the C-4 model
- The solution must provide a README to support easy onboarding
