# Rapid Prototyping & Knowledge Management
Designing, Building, Maintaining, and Improving an Obsidian Plugin Longterm brings the following Challenges upon success:

- Release Management & Distribution
- a growing community of users
- Feedback, Support-, and Feature-Requests, Bug-Reports

## User Voice

- I need a platform for rapid prototyping
- I want to quickly test out new plugin ideas for Obsidian
- I want to use a Starter-Kit to automate Project-Setup and Development-Environment to rapidly test new Ideas
- I want to start my Projects in Obsidian to manage content and knowledge
- I want to follow a Template-First approach to streamline my documentation
- I want an easy to follow Documentation, supporting my day to day tasks
- I want to avoid double or manual work
- I want to Triage feedback and categorize new feedback by Issue
- I want to create a publishable plugin for Obsidian following agile best-practices
- I want to use GitHub as my primary tool for Distribution, Communication, and Collaboration
- I want to use Obsidian as my primary tool for Content Creation and Documentation
- I want to use Git as my primary tool to version my work and connect problem- with solutionspace by linking commits with issues 
- I want to use GitHub as my primary tool for issue tracking
- I want to have a ready to use one click platform, providing me with a development environment consisting of an initialized git repo, issue tracker, a change ,publishing-, and feedback management system
- Despite moving fast and breaking things, I need a stable foundation supporting agile and lean thinking with compliance and regulations in mind
- I want to be prepared for success
