# Workflow
As a User, I have various Ideas for new Obsidian Plugins to improve my Workflows but no coherent System to Track, Manage, Design and Document those.

1. Fork the Starter-Kit Repository
2. Setup and configure the Repository
3. Create the Issues the new Plugin wants to tackle
4. Create a "Proof of Concept" Branch
5. Create a "Proof of Concept" Pull Request in the State: "Draft" pointing to that new Branch
6. Define and Describe the Scope of your first Release in that Pull-Request
6. Git clone the Repo
7. Check-Out the "Proof of Concept" Branch
8. Update Config Files
9. Update README.md
10. Commit and Push with Message: #PR-ID Init
