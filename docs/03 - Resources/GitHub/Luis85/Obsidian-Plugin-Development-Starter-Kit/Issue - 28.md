# Change Management
To change the System one must understand the System.

- How can the process of giving, receiving, and processing Feedback look like to make triage of such Feedback as easy as possible?
- We can receive Feedback in the form of: Requirements, Ideas, Issues, Bugs, Pull Requests, and Comments in GitHub


## Idea

To propose a change, open the Issue Tracker and check if one the top level issues would fit the domain of your idea.

Always start change with a discussion, if your desired domain already exists, begin a new discussion in the corresponding Issues comments.

To propose a Change in form of a Pull Request, you first need to create a new Branch with a describing title what you want to improve. At best you prefix your Pull Request with the domain.

Next, create a new Pull Request in Draft Mode and take your time describing your idea once done, save and ask for review to get feedback and help bring your Idea to Production.
